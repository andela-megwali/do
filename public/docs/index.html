    API Documentation

    Introduction

    Welcome to the Do! API. Do! is a Bucketlist Service that empowers you to keep track of those important things you need to get done in your lifetime. Whether you want to see Paris, climb Everest, learn the piano, start a charity or write a will, Do! helps you keep track and live your dreams while you are alive.

    You can manage your bucketlists through any app or device combination which you authorize. Do! integration is seamless with any app, giving you a highly customisable and perhaps familiar experience.

    This documentation defines the ground rules of engagement with the Do! API in an easily consumable manner. You can checkout our GitHub repository if you want to contribute or raise an issue. Before you kick the bucket, Do!

    Functions and Features

    Below are things you can do with Do! API

    • Create a Do! account
    • Create and manage bucketlists
    • Create items in your bucketlists
    • Track items that have or have not been done/accomplished
    • Search for bucketlist or bucketlist items by name
    • Paginate and limit search queries at will
    • No unauthorized access to your bucketlist
    • Integrates seamlessly with any app
    • Simultaneous access to your account from all your authorized apps / devices
    • Terminate all active access to your bucketlist with one request

    Do! Endpoints

    Do! Endpoint data is returned in JSON format. Whenever a request is made to an endpoint that does not exist, we do our best to handle such scenairos by returning a friendly error message. Below is a list of available endpoints and their corresponding access rights:

    Verb URI Functionality Public Access
    POST /users Creates a new user / account TRUE
    POST /auth/login Logs a user in an assigns an authentication token TRUE
    GET /auth/logout Logs a user out off all active authorized API sessions FALSE
    POST /api/v1/bucketlists Create a new bucketlist FALSE
    GET /api/v1/bucketlists List all the created bucketlists FALSE
    GET /api/v1/bucketlists/:id Get a single bucketlist FALSE
    PUT /api/v1/bucketlists/:id Update the selected bucketlist FALSE
    DELETE /api/v1/bucketlists/:id Delete the selected bucketlist FALSE
    POST /bucketlists/:bucketlist_id/items Creates a new item in the selected bucketlist FALSE
    GET /bucketlists/:bucketlist_id/items Lists all items in the selected bucketlist FALSE
    GET /bucketlists/:bucketlist_id/items/:id Fetches a single bucketlist Item FALSE
    PUT /bucketlists/:bucketlist_id/items/:id Updates a selected bucketlist item FALSE
    DELETE /bucketlists/:bucketlist_id/items/:id Deletes an item in a bucketlist FALSE

    Create User Account

    Create a new user account to get started on Do! Simply send a request to the appropriate endpoint as shown:

    HTTP Request

    POST http://do-bucketlist.herokuapp.com/users

    Query Parameters

    Parameter Description
    user[firstname] Your firstname
    user[lastname] Your firstname
    user[email] Your email
    user[password] Your password

    Authentication

    Login

    To create, manage or interact with your bucketlist, you need to be signed in.

    To login and get an authentication token

    curl -d 'user[email]=we@me.com&user[password]=asdfghj' https://do-bucketlist.herokuapp.com/auth/login

    https://do-bucketlist.herokuapp.com/auth/login?user[email]=we@me.com&
user[password]=asdfghj

    This should return a JSON authentication token

    { 
  "auth_token":"Some12344AuthenTICATIOn78890toKEn"
}

    HTTP Request

    POST https://do-bucketlist.herokuapp.com/auth/login

    Query Parameters

    Parameter Description
    user[email] Your email
    user[password] Your password

    Do! expects that the API authentication key you received is included in all API requests in a header that looks somewhat like one of these:

    { "Authorization" => Some12344AuthenTICATIOn78890toKEn }

    Authorization: Some12344AuthenTICATIOn78890toKEn

    The authentication token has an expiry period of two (2) hours after which you will need to login again for a new token
    You can login on multiple devices and via multiple apps. Each app/device would receive an API authentication token.
    For security reasons, failure to add your authentication token to your requests would result in automatic denial of all such requests

    Logout

    To logout of all active sessions

    curl -H 'Authorization: Some12344AuthenTICATIOn78890toKEn' https://do-bucketlist.herokuapp.com/auth/login/auth/logout

    https://do-bucketlist.herokuapp.com/auth/logout

    This should return a JSON logout message

    { 
  "message":"User logged out of all active sessions"
}

    While tokens are valid for 2 hours, you may want to invalidate all your active tokens when you are done interacting with your account.

    HTTP Request

    GET https://do-bucketlist.herokuapp.com/auth/logout

    Query Parameters

    none

    This action invalidates all your active tokens on all your devices / apps. You would need to login again if you want to interact with your account / bucketlist again.

    page This offsets the returned result to the desired page requested

    The maximum number of Bucketlists or Items you can request per page is 100. If you specify a limit that is greater than 100, we would only return 100 results that match your parameter filter. That’s how we Do!

    Search for bucketlists by name

    To find a bucketlist by name:

    curl -H 'Authorization: Some12344AuthenTICATIOn78890toKEn' 
https://do-bucketlist.herokuapp.com/api/v1/bucketlists?q=Anew

    https://do-bucketlist.herokuapp.com/api/v1/bucketlists?q=Anew

    This should return a JSON response like this:

    [
  {
    "id":1,
    "name":"AnewOne",
    "items":[],
    "date_created":"XXXX-XX-XX 16:32:32",
    "date_modified":"XXXX-XX-XX 16:32:32",
    "created_by":"Mandy"
  }
]

    We do not expect you (or your app) to remember all the id numbers of your bucketlists. To find or filter your bucketlists by name, simply add the query filter parameter (q) along with the bucketlist name as shown. You can give the name in full or just a part of the name. This would return all your bucketlists matching that name or that have your query string in their name.

    HTTP Request

    GET https://do-bucketlist.herokuapp.com/api/v1/bucketlists?q=somebucketlist

    Query Parameters

    Parameter Description
    q This filters your bucketlists by name using the query string parameter
    Do! you know? You can also specify limit and page parameters as previously described to further customise your search results.

    Delete a bucketlist

    To delete a bucketlist with id: 1

    curl -H 'Authorization: Some12344AuthenTICATIOn78890toKEn' -X DELETE 
 http://localhost:3000/api/v1/bucketlists/1

    https://do-bucketlist.herokuapp.com/api/v1/bucketlists/1

    This should return a JSON response somewhat like this:

    { "message":"Bucketlist deleted" }

    If you feel a bucketlist is not worth pursuing anymore and you want to delete it, then just Do! it. No questions asked.

    HTTP Request

    DELETE https://do-bucketlist.herokuapp.com/api/v1/bucketlists/1

    Query Parameters

    none

    Once you delete a bucketlist that you own, it’s gone, forever, along with all the items created under it. We do not keep any backups whatsoever because we respect that you no longer want the deleted buckelist associated with you. Please use this request/command with caution. If you are not sure, don’t Do! this one.
    If you are not totally sure about keeping a bucketlist, perhaps because you are waiting for a trigger event; you can create another bucketlist to serve as a reminder to delete the marked bucketlist if/when such event occurs :)

    Possible Errors

    This is the error explanation section. Check here to identify the cause of any problems you may have encountered while using Do! API.

    Do! API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong, expired or tampered with
    403 Forbidden – The requested resource is not for you (hacking are we?)
    404 Not Found – The specified request could not be found
    405 Method Not Allowed – You tried to access a resource with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The requested resource has been removed from our servers
    418 I’m a teapot :)
    429 Too Many Requests – You’re requesting a lot. Do! not overwork yourself, slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
    That’s all folks, that’s how you Do!
